# CortexLink 联调 Checklist

> 用于各模块集成联调时的逐项验证，覆盖设备、规则引擎、Lua 沙箱、Cron、APP、LLM 等全部子系统。

---

## 1. 环境准备

- [ ] **1.1** Mosquitto broker 远程服务可达，端口通畅
- [ ] **1.2** SQLite 数据库文件可正常创建/读写
- [ ] **1.3** `~/.cortexlink/scripts/` 目录存在且可写
- [ ] **1.4** OpenClaw MCP Server（`127.0.0.1:18789`）已启动并可访问
- [ ] **1.5** LlmSqlProxy（端口 `8899`）已启动，`POST /sql` 接口可访问
- [ ] **1.6** spdlog 日志正常输出（daily rotating sink）
- [ ] **1.7** 主机编译通过，二进制可正常运行

## 2. 设备上线流程

| # | 检查项 | 预期结果 | 结果 |
|---|--------|----------|------|
| 2.1 | 设备连接 broker 后向 `broadcast/online` 发送属性 JSON | 主机收到消息，解析成功 | ☐ |
| 2.2 | 主机将设备属性写入 `device_property` 表 | 表中存在该设备记录，`dev_state = online` | ☐ |
| 2.3 | 主机将设备事件写入 `event` 表 | 表中存在该设备上报的事件记录 | ☐ |
| 2.4 | 主机向 `device/{uuid}/resp/m2s` 回复 `resp: 0`（OK） | 设备收到回复，resp 为 0 | ☐ |
| 2.5 | 已存在设备再次上线（state 已 offline → 改 online） | `dev_state` 更新为 online，不重复插入 | ☐ |
| 2.6 | 设备属性 JSON 格式错误（缺必填字段） | 主机回复 `resp: 4`（INVALID_REQUEST）或 `resp: 5`（MISSING_FIELD） | ☐ |
| 2.7 | 设备注册失败场景 | 主机回复 `resp: 3`（INTERNAL_ERROR） | ☐ |
